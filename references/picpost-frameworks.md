# Picpost Frameworks · 贴图笔记框架库(微信公众号「贴图」+ 小红书互通)

> 5 套贴图框架 P1-P5 · 每套定 N 张图 + caption 写法。
> 一份内容两边发:微信公众号「贴图」tab + 小红书图文笔记。

## 通用规则

### 输出结构(每篇贴图笔记)

```
output/picpost/<YYYY-MM-DD>-<slug>/
  ├── post-1.png ... post-N.png      # 1:1 (1080×1080) · 3-9 张
  ├── caption-wechat.txt             # 公众号贴图文案(简洁 · 150-300 字)
  ├── caption-xhs.txt                # 小红书文案(带 #话题 / 表情 / 200-500 字)
  └── meta.yaml                      # 主题 / 框架 / tags / 私域引流位
```

### 视觉规则

```
画布   · 1080 × 1080 (1:1)
风格   · 复用 case-realistic.md 拟真截图美学(数字 + 时间戳 + 真实感)
首图   · 必有大字标题(像小红书封面那种「!!!」型钩子)
末图   · 必有「关注 / 加私域」CTA(简化版 · 1 个 QR + 一句话)
中间图 · 信息密度高 · 单图传达 1 个核心点 · 不要塞满
```

### 跟长文 cover.png 的差异

| 维度 | 长文 cover (2.35:1) | 贴图 post (1:1) |
|---|---|---|
| 数量 | 1 张 | 3-9 张 |
| 用途 | 文章顶部封面 | 独立可滑动 · 每张要单独成立 |
| 文字 | 主标 + 副标 | 每张一句话核心点 |
| 阅读节奏 | 进文章后看 | 卡片式 · 像翻 PPT |

## P1 · 时间线复盘(对接 case_study)

**用途**:30 天 / 90 天 / 半年复盘 · 每张图代表一个时间节点。

### 张数 5-7 张

```

post-2 · Day 0 起点 · 「我用 X 之前 / Day 0」+ 数据
```

## 不做什么

- ❌ 单篇贴图 > 9 张(微信「贴图」上限就是 9 · 小红书也是)
- ❌ 文字过多塞图(每张单图最多 30-50 字 · 不要小红书化「图当海报」)
- ❌ 用 case-realistic 严肃风做金句卡(P5 应该轻松点)
- ❌ caption-xhs 不带 #话题(小红书无 # 流量减半)
- ❌ caption-wechat 带 #话题(公众号贴图不识别 # · 看着乱)
- ❌ 自动发 API(API 还没公开 · 留 user 手动)
