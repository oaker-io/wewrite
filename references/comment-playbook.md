# Comment Playbook · 评论区运营 SOP

> 公众号涨粉的关键不在「文章质量」 · 在「评论区氛围」。
> 文章下面 0 评 vs 30 评 · 路过用户的关注转化率差 5-10 倍。
> 极客杰尼范文 4 篇都是 11+ 评论 · 87 转发 · 这里学他的运营手法。

## 4 件死守的事

### 1. **作者必抢沙发 + 置顶**

文章群发后**第一条评论必须是作者本人** · 内容是「下一篇预告」或「问题引导」。
然后**置顶这条** · 让所有访客一打开评论区就看到。

为什么:
- 制造「这篇有连载」的预期 · 用户回头看
- 给后续评论一个回答方向(降低用户「不知道说什么」的心理门槛)
- 提升评论数(用户跟着作者的话题走 · 比自由发挥容易)

### 2. **24 小时内回复每一条评论**

哪怕只是一个 emoji(参考极客杰尼回「🤝」)· 每条都回。

为什么:
- 评论者收到「作者赞过 / 作者回复」push 通知 → 二次打开
- 其他读者看到「作者会回」→ 自己也敢评
- WeChat 评论数 + 作者互动数都计入 CES 算法 · 拉曝光

### 3. **每条评论引导下一步动作**

不要只回「谢谢」「同感」 · 每条回复都给一个 CTA:
- 「细节我下篇展开 · 关注一下」
- 「你这个场景挺典型 · 加我微信备注 X · 我详细聊」
- 「这个坑我也踩过 · 在 Day 7 复盘里有完整方案」

为什么:
- 把评论区变成私域引流入口
- 让看评论的人也想加微信(社会证明)

### 4. **每周筛 3 条优质评论 → 做成下篇钩子**

每周日整理本周评论 · 挑出 3 条「最有共鸣 / 最有问题感」的 · 下周写成 S4 读者问答。
S4 副推开头直接「@<昵称> 在群里问我:<具体问题>」 — 评论者本人会被通知到 · 必转发。

为什么:
- 内容有源头(读者真问题 · 不靠瞎编)
- 制造「写到我了」的兴奋感 · 评论者自动当宣传员

## 置顶留言模板库

### 模板 A · 连载钩子(适合干货 / 案例 / 系列文)

```
🐉 【下篇预告】
本文是「<系列名>」第 <N> 篇 · 下一篇:
「<下篇标题>」<2026-MM-DD 大概日期>

关注一下不迷路。
```

### 模板 B · 问题引导(适合教程 / 评测 / 工具)

```
☝️ 【问个问题】
你们用 <工具/方法> · 最大的坑是哪个?
评论区告诉我 · 我下篇统一回答 Top 3。
```

### 模板 C · 资源补充(适合干货)

```
📦 【完整资源】
本文用到的 <清单 / SOP / 模板> · 已整理成 <Notion / Google Doc>。
评论区扣「<关键词>」· 我私发链接。
```

### 模板 D · 社群引流(适合案例 / 复盘)

```
💬 【感兴趣的可以聊】
做这事踩过 N 个坑 · 文章只能写主线 · 细节都在我读者群里。
加我微信备注「<关键词>」· 我拉你进群。
```

### 模板 E · 反向问问题(适合观点 / 热点)

```
🔥 【你怎么看?】
本文观点比较冲 · 我知道有人不同意。
不同意的来评论区辩 · 我每条都回。
```

### 模板 F · 数据补充(适合数据快讯 / 评测)

```
📊 【数据来源】
本文 <N 个数据点> 全部来自:
<source-1>: <url>
<source-2>: <url>
有疑问可以原文核对。
```

### 模板 G · 30 天约定(适合开始连载)

```
📅 【30 天复盘约定】
我接下来 30 天会日更 <主题>。
关注一下 · 30 天后回来看变化。
```

## 自动回复模板库

### 跟「同感 / 学到了 / 有用」类正向评论

回 `🤝` 或 `+1` 或 一个 emoji · 不要套话「感谢支持」(显得敷衍)。

### 跟「我也踩过这坑」类共鸣

回 `这个坑大家都踩 · 我下篇统一总结 · 关注一下` 或类似 — 引导关注。

### 跟「能不能写 X」类需求

回 `已记下 · 周内安排 · 想看 X 的可以扣 1`(收集需求 + 制造期待 + 让其他人也表态)

### 跟「具体怎么做」类技术问题

回 `这个细节比较深 · 加我微信备注「<关键词>」我详细聊 · 评论里说不清` — 引私域

### 跟「不同意 / 你说错了」类反驳

回 `<具体技术回应 1-2 句> · 你的场景比较特殊 · 我私聊跟你聊聊?加我「<关键词>」` — 不对线 + 引私域

### 跟广告 / 引流评论

不回 + 删除(WeChat 后台手动) · 不要让评论区被污染

## 评论运营时间节点

```
T+0    群发出去                                 (用户操作 · 19:30 后)
T+5min comment_kickoff.py 推 Discord 候选置顶语 (auto-comment.sh 触发)
T+10min 用户复制候选语 · 到 mp.weixin.qq.com 评论区粘贴 + 置顶
T+1h   第一波评论涌入 · 用户开始手动回(每条 1-2 句)
T+4h   评论稳定 · 用户睡前再扫一遍未回的
T+24h  整理本周评论(周日做 · 挑 3 条做 S4 读者问答素材)
```

**自动化边界**:置顶留言/回复评论这类 *写入操作* WeChat 公开 API 没开放 ·
全部需要用户手动 `mp.weixin.qq.com` 操作。但**候选话术 / 提醒时机**全自动化。

## 数据指标(每周回顾)

| 指标 | 健康值 | 不达标怎么办 |
|---|---|---|
| 单篇评论数 | ≥ 5 | 置顶留言/钩子句太弱 · 改模板 |
| 作者回复率 | 100% | 24h 内必回 · 用 Discord 提醒 |
| 评论 → 私域转化 | ≥ 1 / 篇 | 引流话术不够具体 · 加「备注 X」 |
| 评论 → 关注转化 | ≥ 5% | 内容不对路 · 调 idea 库选题 |
| 沙发抢到率 | 100% | comment_kickoff.py 时间太晚 · 调早 |

每周日 stats 回填时一并算这几个指标 · 写到 `output/history.yaml`。

## 不做什么

- ❌ 不刷评论(刷评论会被检测 · 降权)
- ❌ 不删差评(除非广告 / 攻击 · 否则保留 · 显得真实)
- ❌ 不在评论区发长文(评论一句话即可 · 长论证去主推)
- ❌ 不用 emoji 堆砌(每条 1-2 个 emoji 即可 · 多了像表情包账号)
- ❌ 不做「评论区 N 楼送」之类活动(短期拉数 · 长期降低真实度)
