# Topic Tags · 公众号话题标签策略

> 微信公众号支持 `#话题` 标签(2026 年 feed 算法用 · 命中话题池流量增加 30-50%)。
> 极客杰尼范文末尾就带 3 个 `#养虾 #养虾新玩法 #OpenClaw` · 是 87 转发的关键之一。
> 本文档给:每篇必带 3-5 个标签 · 主推 / 副推话题对应表 · 标签库。

## 4 个铁律

### 1. 每篇必带 3-5 个 `#话题`

放在文末**最后 1 行**(author-card 前)· 用空格分隔。

格式:
```
#AI #AI Coding #武汉创业
```

为什么 3-5:
- 少于 3 个 · feed 算法不够分发
- 多于 5 个 · 看起来像营销号 · 用户反感

### 2. **1 个固有 #** + 2-4 个**主题相关 #**

**固有标签**(每篇必带):`#AI 非共识`(品牌核心 · 跟「宸的 AI 掘金笔记」绑定)
**主题标签**:根据当篇主题选 2-4 个

### 3. 主推和副推**话题不重复**

主推用「概念性」标签(eg `#AI Coding`)
副推用「具体产品」标签(eg `#Cursor` `#ClaudeCode`)
这样 feed 算法把它们分发到不同池 · 触达不同用户

### 4. 月内不要重复用同一组标签

每周轮换 · 给算法新鲜感 · 否则同一组标签反复用 · 排名会下降。

## 标签生成规则(给 write.py prompt 用)

写文末时按这个顺序选:

```
1. #AI 非共识                          (固定 · 必带)
2. <从主题抽 1 个具体产品/工具>          (eg #Cursor / #Claude)
3. <从内容形式选 1 个>                  (eg #案例复盘 / #保姆级教程)
4. <从领域场景选 0-1 个>                (eg #AI Coding / #独立开发)
5. <地域/品牌选 0-1 个>                 (eg #武汉 AI 创业 / #openclaw)
```

总数控制 3-5 个 · 多了删 4-5 · 少了从领域 / 形式补。

## 标签放在哪

```markdown
... 正文 ...

(空 1 行)

#AI 非共识 #Cursor #案例复盘 #AI Coding

(空 1 行)

:::author-card
...
:::
```

**位置规则**:
- 标签段必须在 author-card **之前**(微信识别 # 是按文中位置)
- 标签段独占 1 行 · 前后空行
- 不要把 # 嵌在正文中(WeChat 不会把正文内的 # 当话题)

## 不做什么

- ❌ 标签里不放 emoji(eg `#🔥AI` · 不识别)
- ❌ 不带空格的复合词(`#AI 非共识` 是对的 · `#AI非共识` 是另一个不同的标签)
- ❌ 不蹭无关热点(eg `#周杰伦` · 跟内容不符 · 算法降权)
- ❌ 不超 5 个(超过看起来像营销号)
- ❌ 不每篇都同一组(月内换 3-4 组)
